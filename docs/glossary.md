# 📖 术语大白话词典

> 这篇文章里没有「技术」，只有「人话」。遇到不懂的词，Ctrl+F 搜一下。

---

## 基础概念

### 终端 / 命令行

**一个黑窗口，用来跟电脑「打字交流」。**

类比：就像微信聊天窗口，只不过你聊天的对象是电脑。你打字输入指令，电脑回复结果。

![终端长这样]()  ← 可以放一张截图

**为什么需要它？** 因为很多操作没有图形界面（App 图标），只能用文字指令控制。Agent 帮你敲这些指令，你不用自己敲。

---

### 代码

**给电脑看的「说明书」。**

你给电脑写一段文字（代码），电脑按照你的意思运行，显示出一个网页、一个 App 或者处理一批数据。

类比：代码就像菜谱。你告诉厨师（电脑）「放 200g 面粉、2 个鸡蛋、烤 30 分钟」，它就做出一个蛋糕。

**你会看菜谱就行，不需要自己写。** Agent 帮你写代码（写菜谱），电脑帮你执行（做菜），你只管吃（看结果）。

---

### Git

**代码的「历史记录 + 网盘」合体。**

它做两件事：
1. **记录每一次改动**（谁改的、什么时候改的、改了什么）— 像 Word 的「修订模式」
2. **把代码同步到网上**（GitLab / GitHub）— 像百度网盘，但是给代码用的

**为什么需要它？** 团队协作时，你不会覆盖别人的代码；改错了可以回退到之前的版本。

---

### GitLab / GitHub

**放代码的「网盘网站」。**

- GitLab：一个网站，你把代码传上去存着
- GitHub：也是放代码的网站，跟 GitLab 差不多（就像百度网盘和阿里云盘的区别）

类比：GitLab = 公司公共网盘，GitHub = 个人网盘。

---

### 仓库（Repository / Repo）

**一个项目的「文件夹 + 历史记录」。**

你在 GitLab 上创建一个「仓库」，就等于创建了一个带历史记录功能的文件夹。你可以把本地的代码传上去，也可以从上面下载别人的代码。

---

## 身份和权限

### Token / 密钥

**一串字母数字组成的密码，用来证明「我是本人」。**

类比：银行卡密码。别人拿到你的 Token 就能以你的身份操作你的代码，所以**绝对不能告诉别人**，也**不能截图发到群里**。

**Agent 怎么用 Token？** Agent 用一个叫 Keyring 的工具帮你存好 Token，每次用的时候从 Keyring 里取，不暴露明文。

---

### 登录 vs 认证

- **登录**：输入用户名密码 → 证明你是你
- **Token 认证**：输入 Token → 也是证明你是你，但更安全（可以随时撤销、可以只给部分权限）

**为什么不用密码？** 密码泄露了就全完了。Token 只能做特定的事情（比如只能看代码不能删），而且随时可以废弃换新的。

## 操作流程

### 克隆（Clone）

**「帮我把网上的代码下载到我的电脑上。」**

从 GitLab/GitHub 上把整个项目的文件下载到你的电脑。

---

### 推送（Push）

**「帮我把电脑上的代码上传到网上。」**

把你电脑上改好的代码同步到 GitLab/GitHub 上，这样别人也能看到。

---

### 提交（Commit）

**「保存我现在的改动，写个备注说明我改了什么。」**

就像 Word 里 Ctrl+S 保存，但同时写一行备注：「把登录按钮颜色改成蓝色」。

---

### 分支（Branch）

**「在代码的副本上改，改好了再合并回去。」**

类比：你在写一份 Word 文档，老板说「试试把第三章改成另外一种风格」。你不会直接在原文上改——你先复制一份，在副本上改，老板看完了觉得 OK，再把副本的内容替换进原文。

这个「副本」就叫分支。

---

### MR / PR（Merge Request / Pull Request）

**「请求把我的改动合并进主版本。」**

你在分支上改完了，发一个 MR/PR 给团队，意思是：「我改好了，你们看看，没问题就合并进去。」

---

## 工具相关

### CLI（Command Line Interface）

**只能在「终端」里用的 App，没有图标。**

就像手机上的 App 有图标可以点，但有些特殊工具只能在黑窗口里用文字控制。Agent 帮你敲这些文字。

**常见 CLI：**
- `git` — 操作 Git 的命令
- `glab` — 操作 GitLab 的命令
- `gh` — 操作 GitHub 的命令
- `npm` — 安装代码包的「应用商店」

---

### npm

**代码界的「应用商店」。**

手机上有 App Store（苹果）和应用市场（安卓），可以下载各种 App。npm 就是代码的应用商店，里面有几十万个别人写好的功能包，你可以一键安装到自己的项目里。

**你只需要对 Agent 说**「帮我装一个 xxx 功能包」，Agent 就知道怎么用 npm 装。

---

### 依赖（Dependency）

**「你的项目用了别人的哪些功能包。」**

比如你要做一个网页，你用到了「按钮组件包」「颜色工具包」「图片处理包」——这些就叫你项目的「依赖」。

类比：你要做一盘宫保鸡丁，需要鸡丁、花生、酱料——这些就是你的「依赖」。

---

### 构建（Build）

**「把源代码变成能直接用的东西。」**

你写的代码（菜谱）→ 构建工具处理 → 变成一个可以直接打开的网页（端上桌的菜）。

**你只需要对 Agent 说**「帮我构建一下」，Agent 就知道怎么跑。
