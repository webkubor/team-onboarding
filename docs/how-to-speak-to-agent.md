# 🗣️ 怎么跟 Agent 说话

> Agent 不是读心术师。你说得越清楚，它干得越好。这篇教你「万能公式」。

---

## 万能公式

```
动作 + 目标 + 位置 + 要求
```

### 拆解

| 部分 | 说明 | 例子 |
|------|------|------|
| **动作** | 你要它做什么 | 帮我创建 / 帮我改 / 帮我查 / 帮我删 |
| **目标** | 对什么东西做 | 一个网页文件 / index.html / 背景颜色 |
| **位置** | 在哪（路径/文件夹） | 桌面上 / /Users/我/Desktop/项目名/ |
| **要求** | 具体改成什么样 | 改成蓝色 / 字体放大 / 加一个按钮 |

---

## 好指令 vs 坏指令

### ❌ 坏指令

> 「帮我搞一下那个页面」

Agent 内心：哪个页面？怎么搞？

> 「颜色不好看，改一下」

Agent 内心：哪个颜色？改成什么色？

> 「报错了修一下」

Agent 内心：什么错？在哪报的？

> 「把它改得好看一点」

Agent 内心：好看的标准是什么？

---

### ✅ 好指令

> 「帮我把 /Users/我/Desktop/my-site/index.html 这个文件的背景颜色改成浅蓝色」

> 「运行 npm run dev 的时候报了这个错：『Error: Cannot find module』，帮我看看是什么原因」

> 「帮我把标题的字体从 16px 改成 20px，加粗」

> 「帮我在 homepage 文件夹里创建一个 contact.html，包含一个表单：姓名输入框、邮箱输入框、发送按钮」

---

## 三个黄金习惯

### 习惯 1：给位置

不要只说「那个文件」，说完整路径或者能识别的位置：

- ✅ 「桌面上的 index.html」
- ✅ 「/Users/我/Downloads/图片文件夹 里的所有图片」
- ❌ 「那个文件」

### 习惯 2：给具体效果

不要只说「改好看」，说具体的：

- ✅ 「标题改成 30px 大小，深蓝色，居中」
- ✅ 「按钮改成圆角，背景用浅绿色」
- ❌ 「改好看一点」

### 习惯 3：给报错信息

遇到报错，**截图发过去**或者**复制完整的错误信息**。不要只说「报错了」。Agent 看错误信息就像医生看化验单——不给它看，它就瞎猜。

## 常见场景的标准句式

### 创建新东西

```
「帮我在[位置]创建一个[什么]，要求：[具体要求]」
```

例子：「帮我在桌面创建一个 index.html，背景浅蓝，标题『欢迎』用大号字体居中。」

### 修改已有东西

```
「帮我把[位置]的[什么东西]改成[什么样的]」
```

例子：「帮我把 /Users/我/Desktop/my-page/index.html 的标题从黑色改成红色。」

### 排查问题

```
「我运行了[命令]，报了这个错：[错误信息]。帮我看看怎么修。」
```

例子：「我运行了 npm run dev，报了 `port 3000 already in use`。帮我看看。」

### 装东西

```
「帮我装[工具名]。装完后帮我验证一下装好了没有。」
```

例子：「帮我装 Git。装完后帮我验证一下能不能用。」

### 问概念

```
「什么是[术语]？用大白话解释，不要用技术术语。」
```

例子：「什么是 Git？用大白话解释，打个比方。」

## 进阶：一次说清楚，避免来回

### 初级（需要来回多次）

你：「帮我改标题」
Agent：「改成什么样？」
你：「改成红色」
Agent：「多大的字？」
你：「20px」
Agent：「放在哪里？」
你：「居中」

### 进阶（一次说完）

你：「帮我把首页标题改成红色、20px 大小、居中。另外背景从白色改成浅灰色。」

**一句话把动作、目标、要求全说了，Agent 一次就能做完。**

---

## Agent 听不懂怎么办？

1. **说人话，不要说术语**：你不懂代码，Agent 懂，让它自己翻译。你说「我想把那个按钮做得能点」，它会自己翻译成代码。
2. **给例子**：说「我想做成像淘宝首页那种感觉」比「做大气一点」好 100 倍。
3. **分步来**：如果一次说不清楚，拆成几步。「先帮我创建文件」「文件创建好了，现在帮我加标题」「标题好了，现在加图片」
4. **截图**：你觉得哪里不对，截图发给 Agent，比打字描述有效得多。

---

## 总结

| 原则 | 解释 |
|------|------|
| 说清楚要什么 | 不要模糊词（"好看""大气"），说具体效果 |
| 说清楚在哪 | 给完整路径或明确位置 |
| 说清楚报什么错 | 截图或复制错误信息 |
| 一次多说点 | 一次给完整需求，减少来回 |
| 不懂就说 | 问 Agent「这是什么意思？大白话解释」 |
